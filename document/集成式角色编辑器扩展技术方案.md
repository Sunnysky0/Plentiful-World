# 集成式角色编辑器扩展技术方案

## 1. 背景与目标

当前工具已具备：
- 按 TAG 加载角色列表、显示头像、编辑角色特质与本地化名称；
- 新增角色与写入本地化；
- DDS 解码显示（含部分格式回退）。

本次扩展目标：
- 在编辑器内渲染当前角色肖像；
- 支持导入肖像（`.png` / `.jpg`），自动等比例缩放并进入“缩放+裁剪”弹框；
- 一键完成：导入资源 → 转换为 `.dds` → 创建/更新 `.gfx` spriteType 定义 → 绑定到角色 portrait；
- 特质从“纯名称文本”升级为“特质列表 + modifier 详情编辑 + 新建特质”。

## 2. 设计原则

- **最小侵入**：尽量复用现有 `Util/server/index.js` 与 `Util/src/App.jsx` 结构。
- **可追溯写入**：所有自动写入文件集中到编辑器生成文件，避免污染大量原始文件。
- **可回滚**：生成文件采用固定文件名，便于人工版本管理。
- **性能优先**：列表场景只加载必要数据，详情场景按需加载 trait 详情。

## 3. 数据与文件约定

### 3.1 资源文件

- 角色肖像输出目录：`gfx/leaders/PW_editor/`
- GFX 定义文件：`interface/PW_character_editor_portraits.gfx`
- 角色定义更新：优先在命中角色定义文件中替换 `portraits` 块；缺失时补写。

### 3.2 特质文件

- 编辑器生成特质定义文件：`common/country_leader/PW_character_editor_traits.txt`
- 现有特质详情来源：
  - `common/country_leader/*.txt`
  - `common/ideas/*.txt`（用于补充 modifier 文本展示与 token 引用）

## 4. 后端架构设计

## 4.1 新增 API

1. `POST /api/characters/:id/portrait/import`
- 输入：`imageBase64`, `mimeType`, `crop`（x,y,w,h,scale）, `targetTag`
- 处理：
  - 解码图片；
  - 按默认分辨率（`156x210`）进行等比例缩放 + 裁剪；
  - 输出 PNG 与 DDS；
  - 写入/更新 `spriteType`；
  - 更新角色 `portraits.civilian.small/large` 指向新 sprite；
  - 清理缓存并返回最新 `portraitUrl`。

2. `GET /api/traits/detail`
- 输出 trait 列表（`id`, `modifiers`, `sourceFile`, `raw`）。

3. `POST /api/traits/upsert`
- 输入：`id`, `modifiers`, `sprite`, `random`
- 处理：在 `PW_character_editor_traits.txt` 新建或替换对应 trait 块。

## 4.2 服务内模块拆分（同文件内函数分层）

- **Portrait Pipeline**
  - `normalizePortraitCanvas()`: 默认分辨率与缩放裁剪计算；
  - `encodePngBuffer()`: 生成 PNG；
  - `encodeDdsBuffer()`: 生成 DDS；
  - `upsertPortraitSpriteType()`: 更新 `.gfx`；
  - `upsertCharacterPortraitRef()`: 更新角色 `portraits`。

- **Trait Pipeline**
  - `loadTraitDetails()`: 解析 trait 与 modifier；
  - `serializeTraitBlock()`: 生成 trait 定义文本；
  - `upsertTraitDefinition()`: 写入/更新编辑器 trait 文件。

## 4.3 依赖规划

- 图片裁剪缩放：优先使用 `sharp`。
- PNG/JPG 转 DDS：
  - 优先使用 Node 可用 DDS 编码库；
  - 若编码库能力不足，则保留 PNG 路径并写入兼容 sprite，同时在文档明确 DDS 转换能力边界。

> 安装约束：所有 npm 安装使用国内镜像 `https://registry.npmmirror.com`。

## 5. 前端架构设计

## 5.1 编辑器区块重构

- “角色信息卡”新增肖像预览区（当前图 + 替换按钮）。
- 点击“导入肖像”后弹出裁剪弹框：
  - 上传文件；
  - 预览框内缩放、拖拽裁剪；
  - 实时显示输出框（156x210）；
  - 确认后提交后端 API。

## 5.2 特质编辑模式

- 左侧/中部展示“角色特质列表”；
- 点击某 trait 后右侧显示：
  - trait id；
  - modifiers（键值表单）；
  - 保存修改；
- 支持“新建特质”：
  - 输入 id；
  - 添加 modifier 行；
  - 提交 `upsert`；
  - 并支持将新 trait 一键追加到当前角色 traits。

## 6. 实施计划（工程计划）

### 阶段 A：后端能力
- 新增本地图片导入、裁剪参数消费、资源落盘、GFX 注册、角色 portrait 绑定。
- 新增 trait 详情读取与 upsert API。

### 阶段 B：前端交互
- 集成肖像预览 + 导入裁剪弹框。
- 集成特质列表、modifier 编辑、新建 trait。

### 阶段 C：联调与验证
- CHI 样本角色完整回归：
  - 导入 `.png/.jpg` 成功；
  - 头像预览即时更新；
  - `.gfx` 与角色定义正确写入；
  - trait 编辑、新建、回写后可重新加载显示。

## 7. 风险与对策

- **DDS 编码兼容性风险**：不同库对 DXT/BC 支持不一致。  
  对策：先实现稳定可写 PNG + 可注册 GFX，DDS 能力按库支持分级启用。

- **角色定义文本替换风险**：HOI4 脚本存在多种排版。  
  对策：优先基于已有 block parser 做最小替换，并保留失败回退与错误提示。

- **trait 语法复杂度风险**：modifier 结构可嵌套。  
  对策：编辑器先支持常用扁平 `modifier = { key = value }`，保留 raw 文本查看与扩展口。

## 8. 与参考资料的一致性说明

- Character 定义、portrait sprite 引用、`recruit_character` 与 role 机制遵循官方 wiki 说明；
- trait 与 modifier 的键值格式、数值语义遵循 Modifiers/List_of_modifiers 文档；
- 本工具中的 trait 编辑器以“可编辑可回写”为核心，优先覆盖高频 modifier。
